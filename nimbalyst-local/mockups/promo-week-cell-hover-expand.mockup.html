4Behaviourrules for wiring it into the 52-week screen

⏱Hover intent, not tooltip

Expands after ~200ms of hover; collapses ~150ms after the pointer leaves cell + card (card is hoverable). Moving along the row never flashes cards open.

📌Click pins

Click (or Enter on a focused cell) pins the card so the merchant can compare against other cells or scroll. Esc / click-away unpins. Only one card open at a time.

🧭Edge-aware placement

The card anchors to the cell's top-left and grows right/down. In the last ~5 week columns it flips to grow leftwards; bottom rows grow upwards. It never covers the item name column of its own row.

⚡Lazy data

Cost ladder + weekly KPIs fetch on first hover per cell and cache for the session; tiles show a light skeleton for the <300ms load. The compact grid stays as fast as today.

🎬Grow-from-cell motion

Card scales from the cell's corner (~160ms, slight overshoot) with a blue focus ring tying cell and card together. Reduced-motion users get a simple fade.

📐Fixed anatomy

Sections always appear in the same order — KPIs, rate strip, tactic, ladder — so merchants build muscle memory. Anything missing (e.g. no digital tactic) renders as a muted “—”, the card never reflows.

🔁Inline reforecast

VLC and base price are editable in the card (card auto-pins on first keystroke). Base price is one value for both store and digital — never set per channel. Reforecast re-runs Units / Sales / AGP with the edits; on confirm the new values become the base for this week → W52. Ladder-derived lines (net, dead-net) recompute live.

🏷Change markers on the grid

After a reforecast the base grid shows tiny C / P badges — solid on the week the change was made, hollow on later weeks running on the new base. Hovering any marked week's card shows the values; nothing else on the grid changes.
