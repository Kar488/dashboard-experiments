5Behaviourrules for wiring it into the 52-week screen

⏱Hover intent, not tooltip

Expands after ~200ms of hover; collapses ~150ms after the pointer leaves cell + card. Applies to promoted and non-promoted cells.

🔀Promo ⇄ No-promo toggle

Every week is scored both ways. A non-promo cell shows ＋ Promo; a promo card shows Set no promo. Toggling flips the cell to the other scored option — promo cells show the tactic card, baseline cells show base + cost only.

🧭Edge-aware placement

Card anchors top-left, grows right/down; last columns flip to grow left, bottom rows grow up. Never covers its own item-name column.

🎨Depth = colour

Shallow yellow, Regular blue-grey, Deep purple — a left colour rail + chip on each cell. Replaces the old ▼/▲/= arrows so depth reads without decoding.

✎Override shows the original

Editing VLC or base price keeps the original value struck through in grey beside the input, and marks the field amber until Reforecast.

📅LY on every line

Tactic + cost-ladder rows each show the grey LY value; new items with no history show “LY —” so the column never looks broken.

🎬Grow-from-cell motion

Card scales from the cell corner (~160ms, slight overshoot) with a blue focus ring. Reduced-motion users get a fade.

🗂No separate week screen

The card’s action buttons (Approve / Set-or-Add promo) do everything the old click-through screen did — so that screen is retired.

🔁Inline reforecast

Base price is one value for both store and digital. Reforecast re-runs Units / Sales / AGP; on confirm the new base carries this week → W52. Ladder-derived lines recompute live.
